# 照片服务接口

照片服务负责存放用户投稿照片的元数据。它不保存图片文件本身，只保存图片链接、地点、描述和提交时间。

默认服务地址：

```bash
http://127.0.0.1:3008
```

## 健康检查

```http
GET /healthz
```

用途：确认服务是否可用。

返回字段：

- `ok`：服务是否正常。
- `photos`：当前未删除照片数量。

## 获取照片列表

```http
GET /api/photos
```

用途：获取可以展示在地球和档案馆里的用户照片。

可选参数：

- `limit`：返回数量，最小 `1`，最大 `200`，默认 `200`。

返回内容是照片数组。每条照片通常包含：

- `id`：照片编号。
- `url`：照片链接。
- `city`：城市名称。
- `city_zh`：中文城市名称。
- `country`：国家或地区。
- `lat`：纬度。
- `lng`：经度。
- `description`：照片描述。
- `signature`：署名。
- `submittedAt`：提交时间。

## 注册照片

```http
POST /api/photos
```

用途：登记一张新的用户投稿照片。

请求体示例：

```json
{
  "url": "https://example.com/dusk.jpg",
  "city": "杭州",
  "city_zh": "杭州",
  "country": "中国",
  "lat": 30.2741,
  "lng": 120.1551,
  "description": "湖边天色慢慢暗下来。",
  "signature": "匿名"
}
```

校验规则：

- 照片链接必须以 `http` 或 `https` 开头，并包含有效域名。
- 城市、国家或地区、坐标不能为空。
- 纬度必须在 `-90` 到 `90` 之间。
- 经度必须在 `-180` 到 `180` 之间。

注册响应不会被缓存。

## 软删除照片

```http
DELETE /api/photos/:id
```

用途：让某张照片不再出现在普通列表中。

软删除不会物理删除数据库记录，只写入删除时间，方便后续追踪和恢复。

如果照片不存在，接口会返回 `404` 和 `photo_not_found`。

删除响应不会被缓存。

## 频率限制

服务端会限制单个访问地址每天的投稿次数。

默认限制是每天 `50` 次，可以通过本地环境变量调整：

```bash
IP_SUBMIT_LIMIT=50
```

## 本地文件

运行时数据库默认是：

```bash
server/data.db
```

这个文件不提交到仓库。真实环境变量文件也不提交到仓库。
