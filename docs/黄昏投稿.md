# 黄昏投稿说明

黄昏投稿用于把用户提供的照片放回夜晚地球。它只记录照片链接和地点信息，不接收本地音频、视频、密钥或数据库文件。

## 页面流程

1. 打开“夜晚档案”。
2. 点击“留下你的黄昏”。
3. 选择城市，填写照片链接、国家或地区、描述和署名。
4. 提交后页面会先尝试写入照片服务。
5. 如果照片服务不可用，页面会把投稿临时保存在本机浏览器里。
6. 投稿成功后，照片会出现在照片档案和地球照片点中。

## 必填内容

- 照片链接：需要以 `http` 或 `https` 开头，并且能公开访问。
- 城市：用于在档案里显示地点，并决定地球上的照片坐标。
- 国家或地区：用于补充照片来源。

## 可选内容

- 描述：写下这张照片里的黄昏。
- 署名：可以写名字，也可以保持匿名。

## 服务端写入

照片服务接收投稿后，会把照片元数据写入本地数据库。
前端会随城市一起提交经纬度，服务端会校验坐标是否在有效范围内。

当前接口：

```http
POST /api/photos
```

服务端会补充提交时间，并把软删除状态保持为空。

## 本地兜底

当前端连接不到照片服务时，投稿不会直接丢失。

页面会把投稿保存在本机浏览器里，并继续展示在当前设备的照片档案中。这个兜底只适合本地预览，不等同于正式服务端入库。

## 不接收的内容

- 本地音频或视频文件
- 真实密钥
- 运行时数据库
- 无法公开访问的本地图片路径
- 与照片投稿无关的大文件

## 验收方式

- 打开投稿弹层，所有输入框应能正常编辑。
- 填入公开图片链接后，提交按钮应有明确反馈。
- 服务端可用时，刷新页面后仍能看到投稿照片。
- 服务端不可用时，当前浏览器里仍能看到刚提交的照片。
- 手机宽度下，投稿弹层不应横向溢出。
