# 验收清单

这份清单用于每次完成一个小步之后快速确认体验没有坏掉。

## 页面进入

- 首页能正常打开。
- 页面没有横向滚动条。
- 顶部操作按钮可以点击。
- 页面刷新后仍能进入夜晚地球。

## 地球视图

- 地球能显示出来。
- 地球可以拖动旋转。
- 城市点、照片点、作家窗灯点能正常出现。
- 悬停或点击点位时，信息不应遮挡主要操作。
- 自动旋转和手动操作不会互相卡住。

## 城市与黄昏

- 当前城市名称能显示。
- 当地时间能显示。
- 日落提示能显示。
- 城市详情面板能打开和关闭。
- 切换城市后，照片胶片能跟着变化。

## 档案馆

- 照片视图能打开。
- 诗歌视图能打开。
- 作家视图能打开。
- 三种视图切换时内容不应错位。
- 关闭档案馆后，页面主操作仍然可用。

## 照片

- 照片胶片能横向浏览。
- 点击照片能打开查看器。
- 查看器里能看到城市、国家或地区、描述。
- 查看器打开后，关闭按钮应获得焦点。
- 关闭查看器后，页面不应留下遮罩。

## 诗歌

- 点击诗歌点或档案项能打开诗歌查看器。
- 标题、作者、地点和正文能完整显示。
- 查看器打开后，关闭按钮应获得焦点。
- 长诗不应撑破弹层。
- 关闭后能回到地球。

## 作家夜窗

- 点击作家窗灯点能打开夜窗。
- 夜窗打开后，关闭按钮应获得焦点。
- 敲门后能出现回应。
- 再次敲门能获得新的短回应。
- 新回应应能被温和播报。
- 告别按钮能关闭夜窗。

## 弗洛斯特夜谈

- 夜谈抽屉能打开。
- 输入问题后能得到文字回应。
- 输入框应限制过长文本。
- 新消息出现后应滚到最新回合。
- 城市、作家、闲聊问题都应有合理回答。
- 回应不应触发任何音频播放。

## 黄昏投稿

- 投稿弹层能打开。
- 必填输入框能正常编辑。
- 提交时按钮应有明确状态。
- 服务端可用时，投稿能进入照片列表。
- 服务端不可用时，本机兜底仍能展示投稿。
- 投稿次数用完时，错误提示应带有恢复时间。

## 照片服务

- 健康检查能返回服务状态。
- 照片列表能返回公开照片。
- 照片列表的 `limit` 参数应限制返回数量。
- 注册照片后，列表能看到新记录。
- 软删除后，普通列表不再显示该照片。
- 删除不存在的照片应返回明确错误。
- 注册和删除响应不应被缓存。
- 频率限制不会影响正常少量投稿。
- 接口冒烟验收应覆盖健康检查、列表上限、JSON 错误、限流和软删除。

## 手机宽度

- 页面不应出现横向滚动条。
- 抽屉和弹层应在屏幕内。
- 按钮文字不应溢出。
- 照片、诗歌、夜窗内容应能滚动阅读。

## 构建体积

- 生产构建应保持地球视图和交互抽屉为独立加载块。
- `globe-runtime`、`geo-runtime` 和 `three-runtime` 应分开输出。
- 构建后应能通过 `npm run build:report` 查看 JavaScript 分片体积。
- 体积预算应保持通过：总 gzip 不超过 `760 kB`，非 Three 单个分片不超过 `520 kB`。
- 如果 `three-runtime` 仍有大块提醒，需要在验收记录中说明。

## 档案数据

- 城市、照片、诗歌和作家编号不应重复。
- 坐标应保持在有效经纬度范围内。
- 照片和作家头像链接应为 `http` 或 `https`。
- 诗歌正文、作家夜窗开场和告别文本不应为空。
- 档案数据变动后应运行 `npm run data:check`。

## 提交前

- 已经跑过相关检查或说明了无法运行的原因。
- 已经跑过 `npm run repo:guard` 或说明了无法运行的原因。
- 档案数据改动已经跑过 `npm run data:check` 或说明了无法运行的原因。
- 服务端改动已经跑过 `npm run server:smoke` 或说明了无法运行的原因。
- 已经确认没有真实密钥。
- 没有本地数据库文件。
- 没有音频和视频文件。
- 没有依赖目录和缓存目录。
- 提交信息使用中文，并准确描述这一步做了什么。
