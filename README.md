# 地球上最后的夜晚

一个把夜晚、城市、照片、诗歌和作家放在同一颗地球上的互动档案。

打开之后会直接进入夜晚地球：城市在地表亮起，黄昏时间缓慢流动，照片、诗歌和作家的窗口围绕不同地点展开。

## 现在可以体验

- 可旋转的夜晚地球视图
- 文学城市、普通城市、照片点、作家窗灯点
- 当前城市的近似当地时间和日落提示
- 城市详情面板
- 城市照片胶片和照片查看器
- 夜晚档案馆：照片、诗歌、作家三种视图
- 诗歌查看器
- 作家夜窗：敲门、回应、告别
- 弗洛斯特夜谈抽屉：城市、作家、闲聊问答
- 黄昏投稿表单：服务端优先，本地兜底
- 简洁照片服务：健康检查、照片列表、注册、软删除、限流

## 项目范围

当前专注于地球、城市、照片、诗歌、作家和投稿体验。

这些内容不作为当前范围：

- 音乐播放器
- 歌单和播放控制
- 语音合成
- 外部视频播放桥
- 本地音频、视频文件
- 本地密钥和环境文件
- 运行时数据库
- `node_modules`
- 原始书籍、索引库、缓存和旧部署包

## 本地启动

更完整的步骤见 [本地运行说明](docs/本地运行.md)。

安装依赖：

```bash
npm install
```

启动前端：

```bash
npm run dev
```

默认访问地址：

```bash
http://127.0.0.1:3000
```

如果端口被占用，开发服务器会自动换到下一个端口。

## 照片服务

投稿流程见 [黄昏投稿说明](docs/黄昏投稿.md)。

照片服务放在 `server/` 目录。

安装服务端依赖：

```bash
cd server
npm install
```

启动服务：

```bash
npm start
```

默认服务地址：

```bash
http://127.0.0.1:3008
```

前端连接服务端时，在本地环境里设置：

```bash
VITE_API_BASE=http://127.0.0.1:3008
```

服务端当前接口：

- `GET /healthz`
- `GET /api/photos`
- `POST /api/photos`
- `DELETE /api/photos/:id`

运行时数据库是 `server/data.db`，不会提交到仓库。

## 迭代原则

这个仓库会持续小步提交。

每一次提交都应该对应一个真实变化：一个功能、一个数据批次、一个界面状态、一个服务端接口、一段文档补充，或者一次可验证的优化。

不做空提交，不做时间填充提交，不用含糊的提交名。提交信息使用中文，直接描述这一小步完成了什么。
