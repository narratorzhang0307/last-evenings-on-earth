# 地球上最后的夜晚 PLUS

一个把夜晚、城市、照片、诗歌和作家放在同一颗地球上的互动档案。

打开之后会直接进入夜晚地球：城市在地表亮起，黄昏时间缓慢流动，照片、诗歌和作家的窗口围绕不同地点展开。

## 体验主线

- 先从一颗可以旋转的夜晚地球进入。
- 在城市、照片、诗歌和作家窗灯之间切换视线。
- 通过城市详情看到黄昏、当地时间和关联照片。
- 在夜晚档案馆里按照片、诗歌、作家检索内容。
- 向弗洛斯特夜谈提问，得到只和城市、作家、夜晚人格相关的文字回应。
- 提交一张黄昏照片，把新的城市记忆放进当前浏览器或照片服务。

## 现在可以体验

- 可旋转的夜晚地球视图
- 文学城市、普通城市、照片点、作家窗灯点
- 当前城市的近似当地时间和日落提示
- 城市详情面板
- 城市照片胶片和照片查看器
- 夜晚档案馆：照片、诗歌、作家三种视图
- 档案搜索结果数量提示，空结果时可以一键清空
- 诗歌查看器
- 作家夜窗：敲门、回应、告别
- 弗洛斯特夜谈抽屉：城市、作家、闲聊问答
- 黄昏投稿表单：服务端优先，本地兜底
- 简洁照片服务：健康检查、照片列表、注册、软删除、限流

## 技术组成

- 前端使用 Vite、React、TypeScript。
- 地球视图使用 Three.js 生态和地理数据渲染。
- 交互入口按需加载，减少首屏等待。
- 档案数据以 TypeScript 常量维护，并通过脚本检查编号、坐标和链接。
- 投稿服务使用 Express 和 SQLite，提供列表、注册、软删除和限流。
- 检查链路覆盖仓库边界、运行范围、档案数据、类型、生产构建、体积报表和服务端冒烟验收。

## 本地启动

更完整的步骤见 [本地运行说明](docs/本地运行.md)。

安装依赖：

```bash
npm install
```

启动前端：

```bash
npm run dev
```

默认访问地址：

```bash
http://127.0.0.1:3000
```

如果端口被占用，开发服务器会自动换到下一个端口。

## 照片服务

投稿流程见 [黄昏投稿说明](docs/黄昏投稿.md)，接口细节见 [照片服务接口](docs/照片服务接口.md)。

照片服务放在 `server/` 目录。

安装服务端依赖：

```bash
cd server
npm install
```

启动服务：

```bash
npm start
```

默认服务地址：

```bash
http://127.0.0.1:3008
```

前端连接服务端时，在本地环境里设置：

```bash
VITE_API_BASE=http://127.0.0.1:3008
```

服务端当前接口：

- `GET /healthz`
- `GET /api/photos`
- `POST /api/photos`
- `DELETE /api/photos/:id`

运行时数据库是 `server/data.db`，不会提交到仓库。

## 性能观察

前端生产构建会把地球视图、档案抽屉、城市详情、照片查看器、诗歌查看器、作家夜窗和弗洛斯特夜谈按交互入口延迟加载。

地球相关依赖当前拆成：

- `three-runtime`：Three.js 运行时，当前仍会触发大块提醒。
- `globe-runtime`：地球视图封装依赖。
- `geo-runtime`：地理路径、D3 和拓扑数据依赖。

后续包体优化优先继续处理 `three-runtime`。

`npm run build:report` 会同时检查一个宽松预算：总 gzip 不超过 `760 kB`，非 Three 的单个 JS 分片不超过 `520 kB`，`three-runtime` 原始体积不超过 `1400 kB` 且 gzip 不超过 `380 kB`。

## 迭代原则

这个仓库会持续小步提交。

每一次提交都应该对应一个真实变化：一个功能、一个数据批次、一个界面状态、一个服务端接口、一段文档补充，或者一次可验证的优化。

不做空提交，不做时间填充提交，不用含糊的提交名。提交信息使用中文，直接描述这一小步完成了什么。

- 补强服务端接口验收和中文说明
